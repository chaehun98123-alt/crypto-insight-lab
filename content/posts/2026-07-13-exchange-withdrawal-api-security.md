  ### 🔐 출금 화이트리스트와 API 키, 왜 매번 강조될까요
  
  - 거래소 자산을 지키는 데 KYC·수수료만큼 중요한 게 **출금 주소 화이트리스트**와 **API 키 권한 관리**입니다.
  - 출금 화이트리스트를 켜두면, 사전에 등록하지 않은 주소로는 출금 자체가 차단되는 경향이 있어 계정 탈취 시에도 자산 유출을 막는 마지막 방어선이 될 수 있습니다.
  - API 키는 발급 시 권한을 세분화할 수 있는데, 이 범위 설정을 소홀히 하면 자동매매 봇 하나가 뚫려도 전체 자산이 위험해질 수 있습니다.
  
  > 화이트리스트와 API 권한 최소화, 두 가지만 제대로 설정해도 상당수의 계정 탈취 피해를 예방할 수 있는 경향이 있습니다.

  ### 🔥 API 키 권한, 이렇게 나눠서 씁니다
  
  1. **읽기전용(Read-Only)**: 잔고·거래 내역 조회만 가능. 포트폴리오 트래커 연동 등에는 이 권한만으로 충분한 경우가 많습니다.
  2. **거래(Trade)**: 매수·매도 주문 실행 권한. 자동매매 봇 연동 시 필요하지만, 출금 권한까지 함께 부여할 필요는 없는 경우가 대부분입니다.
  3. **출금(Withdraw)**: 가장 위험도가 높은 권한. 반드시 필요한 경우가 아니면 발급 자체를 피하고, 부여하더라도 IP 화이트리스트와 함께 사용하는 것이 안전한 경향이 있습니다.
  
  ### 📌 계정 보안 체크리스트
  
  - **출금 주소 화이트리스트 등록**: 자주 쓰는 개인 지갑·거래소 주소만 등록해두고, 신규 주소 추가 시 지연 시간(대개 24~48시간)을 두는 거래소 정책을 활용하세요.
  - **API 키 IP 제한**: 봇을 돌리는 서버의 고정 IP만 허용하면, 키가 유출되더라도 다른 환경에서의 접근을 막을 수 있습니다.
  - **API 키 정기 로테이션**: 3~6개월 주기로 재발급하는 것이 누적 유출 위험을 줄이는 데 도움이 됩니다.
  - **미사용 키 즉시 폐기**: 더 이상 쓰지 않는 봇·서비스의 API 키는 바로 삭제하는 것이 안전합니다.
  
  > ⚠️ 어떤 거래소도 API 시크릿 키나 출금 비밀번호를 이메일·SNS·DM으로 요구하지 않습니다. 이런 요청은 100% 피싱입니다.
  
  ### 💡 요약
  
  > KYC와 수수료 비교도 중요하지만, 실제 자산 손실은 출금 화이트리스트 미설정이나 과도한 API 권한 부여에서 발생하는 경우가 많은 경향이 있습니다. 오늘 계정 설정을 한 번 점검해보시길 권합니다.
